# BookMiro

**把一本书读成会生长的知识图谱。**
</br>

## 概述

**BookMiro** 将书籍转化为知识图谱，并与你的阅读进度同步。它不会一次性把整张图谱抛给你，而是只展示你已经读过的部分——因此永远不会剧透后文。

你只需上传一本书（PDF / EPUB / TXT / Markdown），BookMiro 会：

- 将其切分为有序章节（识别章节标题，无标题时按固定长度兜底）。
- 抽取实体（人物、地点、组织、物品、概念）及其关系，并**与原书使用同一种语言**。
- 随你阅读逐章展开图谱，边读边把新的节点/关系流式补入。

## 最新更新 · 2026-07-18

All Chapters 之前有点太老实了：每个名字都画出来，结果字把图盖得严严实实。这次先把这件事修好，再把长书里那些容易让人累的地方收拾了一遍。

- **图终于能看见了。** 密集概览会先留下能帮你认路的名字，撞在一起的标签直接让开；放大以后，再逐步补回更多名称。搜索到和选中的实体始终保留名字。同一套渲染已经用 5,000 个节点、20,000 条关系的浏览器场景跑过。
- **PDF 现在可以按章节读，也可以按物理页读。** 论文、教材默认按页，原始排版、图片和格式都会保留；小说仍默认按章节。
- **章节识别没那么容易被带跑偏了。** EPUB 会优先读取书内目录；PDF 会结合大纲和页面布局，避免把目录、引用或页码误认成章节。
- **阅读不会再被图谱加载拖住。** 阅读器会先打开，窄分栏也能正常使用；全文搜索放到服务端完成，不用先把整本书下载到浏览器。
- **大图快了，也没有被重新排成一张陌生的图。** 全图保留全部节点和连线，先按熟悉的关系结构完成布局，再冻结位置，操作时不会一直抖动或重算。
- **关系方向一眼能看懂了。** 现在按“来源实体 → 关系 → 目标实体”阅读，箭头就在两个实体之间，不再像三块让人猜顺序的表单。
- **实体类型不再区分大小写。** `concept`、`Concept` 和多余空格写法会归到同一个类型，并使用同一种颜色。
- **抽取失败可以直接补救。** 单个章节失败后可在界面重试；主 LLM 服务拒绝请求时，也可以选择配置 OpenRouter 作为后备。

## 功能特性

- **随阅读展开**：图谱随进度生长，仅显示已读章节；当前章节新出现的节点会闪烁，方便识别"本章新增"。
- **语言锁定抽取**：中文书生成中文的节点、关系标签与类型，绝不出现中英混杂。
- **一键回到原文**：每个节点与关系都保存了原文引用，点击即可跳转到书中对应位置阅读上下文，并高亮该段落。
- **轻松浏览关系（关系走查）**：选中一个人物后，可滚动 / 方向键 / 自动播放依次浏览其全部关系，每条都会在图谱中自动高亮——无需逐条点击。
- **后台 + 流式**：阅读器会立即打开，抽取则在后台按章节继续跑完整本书；后续只增量传输新图谱数据，不会反复下载整张图。

## 快速开始

### 方式一：源码部署（推荐）

#### 环境要求

| 工具 | 版本 | 说明 | 检查 |
|------|------|------|------|
| **Node.js** | 18+ | 前端运行时（含 npm） | `node -v` |
| **Python** | ≥3.11, ≤3.12 | 后端运行时 | `python --version` |
| **uv** | 最新 | Python 包管理器 | `uv --version` |

#### 1. 配置环境变量

```bash
cp .env.example .env
# 编辑 .env，填入你的 LLM API Key
```

**必需的环境变量：**

```env
# LLM API（任意兼容 OpenAI SDK 的接口）
# 图谱抽取会逐章调用 LLM，建议选择性价比较高的模型。
LLM_API_KEY=your_api_key
LLM_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
LLM_MODEL_NAME=qwen-plus
```

> BookMiro 通过直接调用 LLM 在本地完成图谱抽取，无需第三方图谱服务。

#### 2. 安装依赖

```bash
# 一键安装（根目录 + 前端 + 后端）
npm run setup:all
```

或分步安装：

```bash
npm run setup          # Node 依赖（根目录 + 前端）
npm run setup:backend  # Python 依赖（后端，自动创建虚拟环境）
```

#### 3. 启动

```bash
npm run dev            # 同时启动前端 + 后端
```

- 前端：`http://localhost:3000`
- 后端 API：`http://localhost:5001`

单独启动：

```bash
npm run backend
npm run frontend
```

### 方式二：Docker 部署

```bash
cp .env.example .env
docker compose up -d
```

默认读取项目根目录的 `.env`，映射端口 `3000`（前端）/ `5001`（后端）。

### 其他部署方式

Render 部署说明、完整环境变量参考、数据备份说明见 [SELF_HOSTING-ZH.md](./SELF_HOSTING-ZH.md)。

## 工作原理

```
上传（PDF/EPUB/TXT）-> 切分章节 -> 逐章 LLM 抽取（语言锁定）
   -> graph.json（节点/边带 first_episode 与原文引用）
   -> 阅读器按当前章节逐步展开图谱
```
